# llama.cpp over OmniGPU on Linux

OmniGPU integrates with `llama.cpp` through its Vulkan backend. Build
`llama.cpp` with `GGML_VULKAN=ON`, then run `llama-cli`, `llama-server`, or
`llama-bench` through the installed `omnigpu-llama` command.

The launcher restricts Vulkan driver discovery to OmniGPU's ICD, disables
Vulkan layers, runs a bounded `--list-devices` probe, selects an exact reported
`VulkanN` device, and requests full model-layer offload. It exits when only CPU
devices are reported instead of silently running the model on the CPU.

Do not launch it through `sudo`, setuid wrappers, or filesystem capabilities.
Vulkan loaders may ignore driver-selection environment variables for elevated
processes. Run the command as a normal user.

## Build llama.cpp

```bash
sudo apt install build-essential cmake ninja-build libvulkan-dev glslc spirv-headers
omnigpu-build-llama-cpp ~/src/llama.cpp
```

The helper builds dynamic CPU and Vulkan backends plus `llama-cli`,
`llama-server`, and `llama-bench`. Its default output directory is
`~/src/llama.cpp/build-omnigpu-vulkan`.

A source tree can also be built manually:

```bash
cmake -S ~/src/llama.cpp -B ~/src/llama.cpp/build-omnigpu-vulkan \
  -DGGML_VULKAN=ON -DGGML_BACKEND_DL=ON -DGGML_NATIVE=OFF \
  -DGGML_CUDA=OFF -DGGML_HIP=OFF -DGGML_SYCL=OFF -DGGML_OPENCL=OFF
cmake --build ~/src/llama.cpp/build-omnigpu-vulkan --parallel \
  --target llama-cli llama-server llama-bench
```

## Verify the remote device

```bash
omnigpu-llama --require-device OmniGPU -- \
  ~/src/llama.cpp/build-omnigpu-vulkan/bin/llama-cli --list-devices
```

Use `--require-device` with a distinctive substring from the remote device line
to prevent accidentally accepting an unexpected device. The launcher probe has
a 15-second default deadline; override it with `--probe-timeout SEC` when a slow
remote startup requires more time.

## Run inference

```bash
omnigpu-llama --require-device OmniGPU -- \
  ~/src/llama.cpp/build-omnigpu-vulkan/bin/llama-cli \
  -m ~/models/model.gguf -p 'Hello'
```

OpenAI-compatible server, bound to localhost:

```bash
omnigpu-llama --require-device OmniGPU -- \
  ~/src/llama.cpp/build-omnigpu-vulkan/bin/llama-server \
  -m ~/models/model.gguf --host 127.0.0.1 --port 8080
```

Benchmark:

```bash
omnigpu-llama --require-device OmniGPU -- \
  ~/src/llama.cpp/build-omnigpu-vulkan/bin/llama-bench \
  -m ~/models/model.gguf
```

The launcher defaults to `--n-gpu-layers 999`, a numeric value accepted by all
three tools. An explicit `-ngl`, `--gpu-layers`, or `--n-gpu-layers` override
must be a positive integer; zero, negative, missing, conflicting, or excessive
values are rejected to prevent an accidental CPU-only run. `--no-auto-offload`
therefore requires an explicit positive layer count.

Pass launcher option `--device VulkanN` to select one exact reported device.
The launcher also recognizes llama.cpp's own `--device`/`-dev` form, including
comma- or slash-separated `VulkanN` lists, and validates every selected device.
`none`, unknown tokens, duplicate/conflicting device arguments, and a mismatch
between launcher and llama.cpp device selections are rejected. `--skip-probe`
is an explicit diagnostic bypass and still requires a concrete Vulkan device
for inference or benchmarking.

## Scope

This integration routes Vulkan operations generated by `llama.cpp` through the
OmniGPU ICD. It does not add GGUF parsing or an LLM runtime to OmniGPU itself.
Actual model compatibility and performance depend on the Vulkan operations and
features implemented by the OmniGPU guest/host protocol. Run the device probe
and a small model before relying on large-model execution.
